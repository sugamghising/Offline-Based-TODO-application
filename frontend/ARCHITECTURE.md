## Key Architectural Decisions

### 1. **Layered Architecture**

Clean separation of concerns: UI â†’ Service â†’ Repository â†’ Database

**Why?** Easy to test, maintain, and modify individual layers.

### 2. **Offline-First Design**

All writes go to SQLite first, sync happens in background.

**Why?** Instant UX, works without network, reliable.

### 3. **Outbox Pattern**

Queue operations for reliable delivery.

**Why?** Guarantees eventual consistency, handles retries.

### 4. **Version-Based Conflicts**

Simple integer version tracking.

**Why?** Easy to understand, detect, and resolve conflicts.

### 5. **Singleton Services**

Database and sync engine are singletons.

**Why?** Single source of truth, shared state.
